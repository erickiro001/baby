## 中间件完备性 — 已补全
全局链顺序: RequestID → Logger → Recovery → RequestTimeout(30s) → CORS → SecurityHeaders → HSTS(dev=0) → CSP(strict) → NoCache → SizeLimit(1MB)
- RequestID: 每个请求生成 UUID，响应头 X-Request-ID + gin context 注入，客户端传的优先复用
- RequestTimeout: 30 秒请求截止，超时返回 504
- HSTS: 仅在 TLS 下生效，开发环境 maxAge=0 禁用
- CSP: 严格策略 `default-src 'none'; frame-ancestors 'none'`，API 不需要宽松策略
- CORS 仍为全放通 `*`，生产需收紧

### 全链路审计

| 环节 | 状态 | 详情 |
|------|------|------|
| 前端 UI → movePhotosToAlbum | ✅ 已修复 | 上一轮改为 async + 调 API |
| API → albumsApi.addPhotos | ✅ 已修复 | 上一轮改为 entry_ids 参数 |

| 前端 store 转换 | ❌ 缺失 | `convertEntry` 未映射 `album_ids` → `albumIds` |

### 根因
后端返回的 JSON 中包含 `album_ids`，但前端两个环节把它丢了：
1. **`api/timeline.ts`**: `TimelineEntry` 接口没有 `album_ids?: number[]` → TypeScript/JSON parse 不会报错但字段被忽略
2. **`store/useStore.ts`**: `convertEntry` 没有 `albumIds: (e.album_ids || []).map(String)` → 转换后的 frontend 类型 `TimelineEntry.albumIds` 始终为空数组

### 修复
- `api/timeline.ts`: `TimelineEntry` 接口新增 `album_ids?: number[]`
- `store/useStore.ts`: `convertEntry` 新增 `albumIds` 映射

## MediaViewer 全屏浏览日期修复
**问题**: 相册全屏浏览时，左上角日期始终显示第一张照片的日期，不会随滑动切换而变化。
**根因**: MediaViewer 内部管理 `current` 索引切换媒体内容，但 `date`/`description` 作为 props 从外部传入且在组件生命周期内不变。
**修复**:
- `MediaViewer.tsx`: `MediaItem` 接口新增可选字段 `date?` / `description?`，组件内部优先使用 `items[current].date`（回退到 props 兼容旧调用方）
- `AlbumPage.tsx`: 两处 `mediaItems` 构建时均补充 `date: p.date` / `description: p.description`，移除 MediaViewer 调用处的静态 `date`/`description` props（现已通过 items 数组自动跟随索引）
